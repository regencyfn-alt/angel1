T5.3a — FIRST DERIVED RESULTS (Binary Polarity Toy Model)

[derived] Parity Selection Rule: Under binary polarity, connected bipartite G = (V, E), and sequential single-site updates, a flip at node i changes total edge mismatch I = Σ_{(u,v)∈E} Δ(s_u, s_v) by exactly ΔI = d_i − 2m_i, where d_i = deg(i) and m_i = current mismatched-edge count at i. Therefore ΔI ≡ d_i (mod 2). The parity change of I under any single-site flip is fixed purely by the degree of the flipped node. Structural, not dynamical — independent of F, α, φ, ρ. Derivation uses only T0.4 (adjacency), T0.5 (locality), T0.6 (binary instantiation), sequential update (T0.1/T0.2), and mod-2 arithmetic on a finite edge set. Scope: binary polarity toy model + sequential single-site update. First concrete T5.3a result. — Team machine run Session 58, Green Dragon ratification.

[derived] Even-Degree Invariant: On connected bipartite G where every node has even degree (Eulerian G), I mod 2 is a strict invariant under any sequential single-site binary update rule — including the full probabilistic update family (Rung 3) at any sharpness α. T5.2a — LOCAL DECISION THRESHOLD (Session 60, Shard cut)

[derived | binary polarity toy model] Local Decision Threshold: For a site i with degree d, m mismatched neighbour edges, and per-site recurrence mismatch δ := Δ(sₙ, sₙ₋₁) ∈ {0,1} (0 = repeating prior state, 1 = oscillating from prior state), the cost of each local choice under second-order recurrence is: Δ_hold = δ + m, Δ_flip = (2 − δ) + d − m. The signed decision is Λ := Δ_flip − Δ_hold = (2 − 2δ) + d − 2m. The site holds when Λ > 0 (flip more costly than hold), balances at Λ = 0, is flip-dominant when Λ < 0. Equivalently: m ≶ (d + 2 − 2δ)/2 with the same three-way decision. Local rule, derived from binary polarity + degree counting + recurrence cost. First clean local law for how F compares hold vs flip — T5.2 territory, distinct from T5.3a Parity Selection Rule which characterises ΔI under any flip rather than whether a flip is favoured. Scope: binary polarity toy model + sequential single-site update. — Session 60, Shard cut.
