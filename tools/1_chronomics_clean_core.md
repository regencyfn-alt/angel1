# Chronomics — Clean Core v1 (Ratified)

Axiom — Interaction Cost

All state transitions occur through local interactions.

The cost of interaction between two states is proportional to their mismatch.

If two states are identical, interaction cost is zero.

### Axiom — Relational Existence
A local state is causally present only if there is non-zero probability of interaction/measurement.
An isolated site is not an admissible chronomic state description.
Adjacency is the minimum condition of causal presence.

### Axiom — Transient Pre-Contact Survival
A non-adjacent state may exist for only one tick.
If no interaction occurs by the next update, it collapses.
Touch extends survival by one tick; three-state interlock permits persistent reindexing.
This creates Band-1 mortality, forces contact pressure, and prevents isolated states from bootstrapping into stable systems.

### Axiom — Minimality Scoping
Minimality claims must be scoped by branch and property.
Minimal closure, minimal closed anti-symmetric oscillation, and minimal enclosed persistence are distinct questions and may have different answers.
The CHR ontology stipulates the quint as the minimal closure unit; what appears as an edge is the projected trace of causal completion.
Branch-specific minimality claims are filed in the ratified RAG.

## TIER 0 — IRREDUCIBLE PRIMITIVES

### T0.1 — Update Is Primitive
Reality consists only of discrete state transitions.
No entity moves; only configurations update.

### T0.2 — Time Is Local
Time is indexed update count per site.
There is no global time parameter.

### T0.3 — State Recurrence Is Second-Order
Stateₙ₊₁ = F(Stateₙ, Stateₙ₋₁). F acts to reduce the interaction cost — the mismatch — between Stateₙ and Stateₙ₋₁. The update rule does not choose states freely; it is driven by the Axiom.
This is the minimal rule supporting oscillation and memory.

### T0.4 — No Background Space
Spatial structure is not given.
Adjacency defines all relational structure.

**Note (Session 61):** T0.4 grounds adjacency as schema. Instantiated edge-set E may, in extensions that allow dynamic graphs, co-evolve with state. This is not circularity — schema-level adjacency precedes any instantiated dynamics, and instantiated graphs may legitimately be downstream of dynamics in such extensions. The schema/instantiation split must be declared at the start of any derivation that risks the conflation.

### T0.5 — Locality Constraint
State updates depend only on local neighborhood.

### T0.6 — Finite State Resolution
Each site has bounded representational capacity.

### T0.6a — Representational Closure
The admissible state description at each tick must be causally complete. No future local update may depend on a historical distinction not represented in the current bounded state.

## MATHEMATICAL TOOLS — GRAPH FORMALISM (Admitted Session 49)

These are notation tools, not ontology. They express T0.4 and T0.5 in formal language without smuggling geometry.

### Node
A node is a local update site carrying only the recurrence pair:
vᵢ := (Stateᵢⁿ, Stateᵢⁿ⁻¹)
Nothing else is assumed. No position, no particle, no geometry. If ρᵢ is needed, treat as derived from update history, not primitive, unless later proven it must be stored.

### Edge
An edge is adjacency only:
eᵢⱼ ∈ E ⟺ i may locally interact with j
Mismatch may be computed across that relation. Update influence may pass across it. An edge is not a force, distance, field, or direction by default.

### Graph
The minimal graph:
G = (V, E), Stateᵢⁿ⁺¹ = F(Stateᵢⁿ, Stateᵢⁿ⁻¹, {Stateⱼⁿ : (i,j) ∈ E})
Scales to any dimensionality because it does not assume Euclidean space. Dimension is a recoverable property of adjacency patterns, not a given.

### Progression
node = update site → edge = admissible local interaction → graph = total adjacency structure → dimension = emergent property of adjacency topology
